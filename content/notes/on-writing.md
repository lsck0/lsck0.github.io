Some principles I keep coming back to when writing.

## Clarity Over Cleverness

> The great enemy of clear language is insincerity.
> --- George Orwell

Write to be understood, not to impress. If a simpler word works, use it. If a sentence can be shorter, make it shorter.

## Structure

- **Lead with the point.** Don't build up to it.
- **One idea per paragraph.** If you're covering two ideas, split.
- **Use headings.** They let readers scan and skip.
- **Use lists** when order or enumeration matters.

## Technical Writing Specifically

1. Define terms before using them.
2. Give examples immediately after abstractions.
3. Use consistent notation throughout.
4. Prefer concrete over abstract: "the function returns 42" beats "the function returns a value."

## On Editing

First drafts are for getting ideas down. Editing is where writing actually happens. Read your work aloud: if you stumble, the reader will too.

Cut ruthlessly. If removing a sentence doesn't lose meaning, it wasn't carrying its weight.
